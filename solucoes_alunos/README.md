# 🦉 Soluções dos alunos - OwlBI

Bem-vindos à área colaborativa do repositório da OwlBI! Este espaço é dedicado a vocês compartilharem suas resoluções dos exercícios propostos durante nossas aulas de preparação para a OBI.

Ao enviar seu código, você não apenas documenta o seu progresso, mas também ajuda outros colegas que possam estar com dúvidas na mesma lógica!

## 📂 Como as soluções devem ser organizadas

Para manter o repositório limpo e organizado, siga a estrutura de pastas abaixo. Crie a pasta caso ela ainda não exista:

```text
solucoes_alunos/
├── aula_01/
│   ├── seu_usuario_do_github/
│   │   ├── nome_do_problema_1.cpp
│   │   └── nome_do_problema_2.cpp
├── aula_02/
│   ├── seu_usuario_do_github/
│   │   ├── fliper_obi.cpp
│   │   └── volume_da_tv.cpp
└── 
```

## 🛠️ Passo a passo para enviar sua solução (Pull Request)

1. Faça um Fork deste repositório para a sua conta do GitHub.
2. Clone o repositório forkado para a sua máquina local:

```bash
git clone [https://github.com/SEU-USUARIO/nome-do-repositorio.git](https://github.com/SEU-USUARIO/nome-do-repositorio.git)
```

3. Crie uma nova branch com um nome descritivo (ex: solucoes-aula-01-seu-usuario):

```bash
git checkout -b solucoes-aula-01-seu-usuario
```

4. Adicione seus códigos .cpp seguindo a estrutura de pastas mencionada acima. Importante: certifique-se de que seu código compila e resolve o problema corretamente antes de enviar!

5. Faça o commit das suas alterações:

```bash
git add .
git commit -m "feat: adicionando soluções da Aula 01"
```

6. Envie as alterações para o seu fork:

```bash
git push origin solucoes-aula-01-seu-usuario
```

7. Acesse a página do repositório original da OwlBI no GitHub e clique em "Compare & pull request".

## 💡 Boas práticas para o seu código
- Comente seu código: se a lógica for complexa, adicione comentários explicando o porquê de usar determinada estrutura.

- Nomenclatura: evite nomes genéricos para as variáveis. Ao invés de a e b, prefira nomes que indiquem o que elas armazenam.
